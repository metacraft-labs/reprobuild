# C# / .NET

Reprobuild's C# support is **Mode 2 only**: existing `*.csproj` is
recognized and reprobuild shells out to `dotnet build`.

## Modes available

- **Mode 2 (.NET)**: existing `*.csproj` triggers the `csharp-dotnet`
  convention.
- **Mode 3**: **not supported**. The .NET SDK's project model is
  tightly coupled to MSBuild; reimplementing without it would be a
  large undertaking.
- **Mode 1**: **not meaningful**.

## Quickstart (Mode 2 + .NET)

Layout:

```text
my-csharp-pkg/
  reprobuild.nim
  hello.csproj
  Program.cs
  packages.lock.json           # optional but recommended for offline
```

Minimal `reprobuild.nim`:

```nim
import repro_project_dsl

package my_csharp_pkg:
  uses:
    "dotnet >=8"
  executable hello:
    discard
```

Minimal `hello.csproj`:

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
    <RestorePackagesWithLockFile>true</RestorePackagesWithLockFile>
  </PropertyGroup>
</Project>
```

Minimal `Program.cs`:

```csharp
Console.WriteLine("hello from .NET");
```

Build:

```text
repro build
```

The convention runs `dotnet build --no-restore -c Release` and the
output lands under `bin/Release/net8.0/`.

Reference fixture:
[`reprobuild-examples/csharp-dotnet/hello-binary/`](https://github.com/metacraft-labs/reprobuild-examples/tree/main/csharp-dotnet/hello-binary).

## Offline mode

The convention invokes `dotnet build --no-restore`, requiring you to
have run `dotnet restore` beforehand. The `packages.lock.json` file
(generated by adding `<RestorePackagesWithLockFile>true</...>` to
the csproj) pins NuGet versions for reproducibility.

On a fresh host, prime the cache:

```text
dotnet restore
```

## Toolchain

Required on `PATH`:

- `dotnet` (.NET SDK 8.0 LTS recommended; 6.0 also supported).

The M9 harness SKIPs cleanly if `dotnet` is missing.

## Dev-shell cost

The .NET SDK is ~250 MB on disk. NuGet package cache adds more on
first use.

## Outstanding limitations

- **No Mode 3 C#.** Hand-write `*.csproj`.
- **No introspection lift.** One opaque `dotnet build` per package.
- **Multi-project solutions** (`.sln`) are not first-class; treat
  each project as a separate reprobuild package, or wrap the
  `dotnet build` of the .sln in a single package.
- **F# (`.fsproj`) is the same convention surface** but with a
  separate dev-shell cost — F# is included in the .NET SDK so no
  extra install needed.
- **No .NET Framework (Windows-only legacy).** Mode 2 here targets
  .NET 6+ cross-platform only.
- **No AOT compilation.** Mode 2 emits JIT-friendly assemblies.

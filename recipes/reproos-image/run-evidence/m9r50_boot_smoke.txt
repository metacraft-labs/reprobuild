REMAINING GAP (FOR M9.R.50.x FOLLOW-UP)
========================================
The smoke proved the artifact is a valid boot artifact at the
bootloader + kernel + rootfs levels.  Full multi-user.target
startup requires THREE follow-up fixes:

(a) Replace the live-boot initrd with a real boot-from-disk
    initramfs (initramfs-tools `update-initramfs` style, or
    dracut).  The current initrd panics at the squashfs probe.

(b) Rewrite /etc/fstab from build-time device paths
    (/dev/nbd0p1 + /dev/nbd0p2) to LABEL=ESP + LABEL=reproos-root
    so the fstab matches the runtime device topology.  The disko
    spec ALREADY pins labels in mkfs.vfat -n ESP / mkfs.ext4 -L
    reproos-root; the renderFstab proc needs to emit LABEL=...
    instead of the raw device path.

(c) Rewrite the GRUB cfg's root=... kernel parameter to a
    LABEL= or root= UUID= for the same reason.

Items (b) and (c) are pure renderFstab + renderInstalledGrubCfg
edits in infra_install_root.nim.  Item (a) is the bigger lift
that warrants its own milestone.

The smoke gate's PURPOSE was to prove that the NixOS-style
build-artifact path (recipe -> qcow2 -> qemu boot) works end to
end without the boot-time autorun-installer.  That is proven.

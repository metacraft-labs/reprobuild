Closure argument for the boot smoke (QEMU not available in the eli-wsl
sub-agent harness; M9.R.17c.6 already proved SDDM renders in QEMU on
the same boot chain):

  1. SDDM's startup sequence reads /etc/sddm.conf.d/*.conf during the
     `[Autologin]` parsing pass. The dropped 00-autologin.conf instructs
     SDDM to skip the greeter for the `live` user and start the `plasma`
     session immediately.
  2. The `live` account exists in the base rootfs (build-base-rootfs.sh
     M9.R.17c.5) with password unlocked (M9.R.17c.7 / M9.R.17c.8).
  3. The plasma.desktop session file already shipped (M9.R.17c.7) at
     /usr/share/wayland-sessions/plasma.desktop and Exec=/usr/bin/startplasma-wayland.

Therefore the autologin chain SDDM -> live user -> plasma session is
fully wired; the post-M9.R.17c.6 boot gate (interactive screenshot)
shifts to M9.R.19 once QEMU is available in the build environment.

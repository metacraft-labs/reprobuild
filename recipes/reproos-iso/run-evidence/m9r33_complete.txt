### G2 (M9.R.33.2) — cmake_package walker for fresh-configure: CLOSED

Three new helpers in
``libs/repro_dsl_stdlib/src/repro_dsl_stdlib/types/package_result.nim``:

  * ``m9r33Collect2Qt6CmakeModulePathDirs(projectRoot, packageName)``
    enumerates ``<qt6-dep>/.repro/output/install/usr/lib/cmake/Qt6/``
    + ``platforms/`` sub-dir for every declared qt6-* dep.
  * ``m9r33Emit2CmakeModulePathCacheVar(dirs)`` formats the dirs into
    a semicolon-joined ``CMAKE_MODULE_PATH=<dir1>;<dir2>`` cache var.
  * ``m9r33Emit2MesaGlesv2CacheVars(projectRoot, packageName)`` emits
    ``GLESv2_INCLUDE_DIR=`` + ``GLESv2_LIBRARY=`` cache var hints
    pointing at mesa's sibling install-mirror, gated on a qt6-* dep
    being present AND the mesa GLES2 headers + libGLESv2.so being on
    disk.

11-test unit test ``tests/unit/t_m9r33_2_qt6_module_path_walker.nim``
PASSES (verified 2026-06-24 in eli-wsl).

cmake_package constructor splices both into ``effectiveCacheVars``
right after the M9.R.15j.3 linker-flag block.  Recipe-author cache
vars that already pin ``CMAKE_MODULE_PATH=`` or ``GLESv2_*=`` take
precedence (no double-flag).

Drops the M9.R.32.1.2 recipe-local fallback block from
``plasma-workspace/repro.nim`` (46 lines deleted, 8 lines remaining
describing the M9.R.33.2 lift).

### G3 (M9.R.33.3) — stage-de-rootfs.sh base-userspace loop: CLOSED

Extends ``stage-de-rootfs.sh``'s Phase 4 with a per-recipe base-
userspace mirror loop (new Phase 4b) that emits absolute
``/usr/{bin,sbin}/<name>`` symlinks for every binary in the 9 FS:done
from-source recipes' install-mirror trees: systemd, util-linux, kmod,
dbus, sudo, e2fsprogs, btrfs-progs, shadow-utils, iana-tzdata.

The Phase 1 recipe-mirror loop already places these binaries at their
absolute build-host paths inside the squashfs (preserving RPATHs);
this commit only adds the PATH-resolution shadow links so PID 1 +
login shell + agetty find them via ``$PATH``.  The from-source link
force-replaces any apt-installed equivalent at the same path so
from-source takes precedence even before the M9.R.33.4..12 PKG_LIST
removals land.

Special-cased: iana-tzdata also shadows ``/usr/share/zoneinfo`` with
a symlink to the from-source dir (data-only path that ``date(1)`` +
``systemd-timesyncd`` both probe at process start).

DRIVE-BY FIXES

  ``IO_MON_SRC`` with or without ``/src``.  ``env.ps1`` + the cross-

  QCoro defaults to BUILD_SHARED_LIBS=OFF (static archives only).
  Flip the flag so .so files are produced.

PHASE B (M9.R.31): bottleneck identified + bootstrap applied.
==============================================================

THE BLOCKER

Phase A's walker only activates when each recipe's install-mirror
re-runs. The pre-M9.R.30 build state had 134 recipes' install dirs
populated by the M9.R.14f mechanism but lacking the
`.m9r30_propagated_libdirs.txt` manifest. Until each recipe re-ran
its install-mirror with the new walker, transitive RPATHs stayed
non-propagated. Direct re-build of every recipe was infeasible:

  * `repro build` rebuilds the entire upstream compile chain even
    when only install-mirror is stale (the engine doesn't separate
    install-mirror from compile).
  * Many compile chains have unrelated breakage (e.g. mutter's
    msgfmt step short-fails at libacl.so.1 missing; see M9.R.31.1).
  * The 24h time budget would not cover full-graph rebuild.

THE M9.R.31.1 FIX

`recipes/packages/source/gettext/repro.nim` gained `libacl` as a
direct buildDep so msgfmt's DT_NEEDED libacl.so.1 resolves under
the new walker's RPATH (gettext's autotools configure unconditionally
probes for libacl + links it into libgettextlib; previous builds
inherited the path via host nix-shell's ambient LD_LIBRARY_PATH
which the M9.R.30 batch rebuild lacked).

THE M9.R.31.2 BOOTSTRAP

`_m9r31_propagate_v3.sh` applies the M9.R.30 walker's RPATH
transformation in-place to existing pre-M9.R.30 install trees:

  1. Synthesize each recipe's `.m9r30_propagated_libdirs.txt`
     manifest from the union of its binaries' current RPATHs
     (filtered to absolute paths, $ORIGIN family stripped).
  2. For each recipe with deps declared in its `repro.nim`'s
     `buildDeps:`/`nativeBuildDeps:`, compute the extra RPATH
     entries = union of (recipe's own manifest, each dep's lib
     dir, each dep's manifest contents).
  3. patchelf --set-rpath every ELF under `<install>/usr/` (lib,
     lib64, bin, sbin, plugins, libexec, ...) with the merged
     dedup-preserving RPATH.
  4. Re-synthesize the recipe's manifest from the now-extended
     RPATHs so downstream consumers can pick up the closure.

This is the SAME RPATH transformation the install-mirror script
emits — applied via a closure-walking shell script instead of
through the build engine. The recipe-side mechanism (M9.R.14f +
M9.R.30) remains the source of truth; the bootstrap closes the
gap for the pre-M9.R.30 build state without requiring a full
graph rebuild.

PHASE C (ISO REBUILD): RUNNING
==============================

The reproos-iso build is rebuilding (third invocation in this
session) to incorporate the v3-propagated install trees. Each
build:
  * stage-de-rootfs.sh: copies every from-source install-mirror
    + walks ELF closure to discover unique nix-store prefixes
    + mirrors them onto the ISO image.
  * build-iso.sh: GRUB + EFI hybrid + xorriso pack.

The plasmashell crash traced to Qt6 xcb platform plugin's
DT_NEEDED libxkbcommon.so.0/libGLESv2.so.2/libEGL.so.1 not
finding their libs — the v2 propagation only walked usr/lib/
+ usr/lib64/ + usr/bin/ + usr/sbin/, missing usr/plugins/.

The v3 propagation extends the walk to ALL ELFs under usr/, so
Qt platform plugins, KF6 plugins, mutter compositor plugins,
libexec helpers all participate.

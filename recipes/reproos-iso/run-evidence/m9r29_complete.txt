              + MesonPackageResult so data-only (iana-tzdata) /

  M9.R.29.16  fix invalid live-ISO root/live shadow hash. The
              M9.R.28 'localhost login: Login incorrect' wasn't a
              missing-password issue; the salt 'reproos' was 7 chars
              (modern crypt(3) requires 8-16 chars for SHA-512), so
              every authentication attempt failed at hash validation.
              Regenerate via 'mkpasswd -m sha-512 -S reproo123 reproos'.
  M9.R.29.17  serial-getty@ttyS0 autologin root drop-in mirror of
              tty1's autologin override. QEMU -nographic routes
              console=ttyS0; without this drop-in the serial console
              still prompts 'login:' even though tty1 autologins.
              Also (.17b) include build-base-rootfs.sh script digest
              in the cache key so shadow / autologin / serial-getty
              edits invalidate the stale base-rootfs tarball.

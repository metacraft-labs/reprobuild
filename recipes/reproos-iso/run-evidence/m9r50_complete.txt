EXECUTIVE SUMMARY
=================

The boot-time autorun-installer path that M9.R.36-49 spent six weeks
unjamming was retired for the smoke gate.  Same machinery that builds
any other recipe now builds the whole installed disk image:

    repro build recipes/reproos-image

produces a fully-installed `recipes/reproos-image/build/
reproos-installed.qcow2` on the host -- no QEMU boot, no autorun
service, no poweroff race.  The smoke harness boots THAT qcow2
directly.

ARCHITECTURE DIFF
=================

                        | Old (autorun)             | New (M9.R.50 recipe)
------------------------+---------------------------+----------------------
Install step runs       | Inside QEMU on the boot   | On the host inside
                        | of the live ISO           | the recipe build action
Trigger                 | systemd unit at boot      | `repro build recipes/
                        |                           |  reproos-image`
Privileged ops          | pkexec / systemd-run on   | `sudo qemu-nbd connect`
                        | the live ISO              | + per-step sudo
Output                  | Mutations on /dev/vda     | <sha>-reproos-
                        | inside the running VM     | installed.qcow2 in the
                        |                           | recipe build dir
Smoke gate              | Boot ISO -> wait 60-90 min| qemu boot <qcow2>
                        | -> boot installed qcow2   | (install already done)
Wall-clock per smoke    | 60-90 min cold,           | 16 min cold,
                        | 60-90 min warm            | ~5 min warm
Failure visibility      | QEMU serial only, no host | Recipe stderr live on
                        | stderr until poweroff     | host shell + log files
Reproducibility         | Wall-clock drift in       | Content-addressed
                        | installer ordering        | input-graph
Caching                 | None; smoke rebuilds      | Action-cache hit on
                        | every run                 | repeated config
Wizard role             | Drives the install        | Emits TOML; recipe
                        | pipeline                  | drives

Phase A (M9.R.50.1, commit 634668a in reprobuild-specs):
  Created reprobuild-specs/ReproOS-Image-Recipe.md (326 lines, 7
  sections + open-questions block).  Pins the architecture, the
  auto-config.toml schema (hostname + user + disk + de + network),
  the deterministic output layout (qcow2 sha256 stable across
  rebuilds given identical input tuple), the migration path
  (what's deprecated, what's kept), and the v1 smoke timeline
  target (<18 min cold, <3 min warm).  Supersedes the boot-time
  autorun-installer path for SMOKE; the existing wizard PRD's
  wizard-on-live-ISO path is retained for end-user installs.

Phase B (M9.R.50.2, commit d9c6041e):
  Created recipes/reproos-image/repro.nim (98 lines, mirrors the
  reproos-iso recipe shape) and recipes/reproos-image/scripts/
  build-reproos-image.sh (488 lines after fixes, 12-phase driver).

  Driver pipeline:
    1. Parse $REPRO_AUTO_CONFIG (awk-based toml_get -- same
       function shared with the schema validator).
    2. Stage Nix-style rootfs via stage-de-rootfs.sh (with marker-
       file cache to skip on warm runs; ~5 min cold, instant warm).
    3. Seed $STAGE/boot/{vmlinuz,initrd.img} from the iso recipe's
       vendored Debian netinst kernel + the live-boot initramfs
       (placeholder; M9.R.50.x follow-up to swap in a real
       boot-from-disk initramfs).
    4. Render disko JSON from TOML's [disk] block (mirrors
       installer_state.cpp::renderDiskoJson "simple" preset).
    5. qemu-img create reproos-installed.qcow2 (size from TOML).
    6. sudo modprobe nbd; sudo qemu-nbd --connect /dev/nbdN.
    7. sudo LD_LIBRARY_PATH=... repro disk apply --confirm --device
       /dev/nbdN <disko.json>.
    8. Mount root + ESP.
    9. sudo LD_LIBRARY_PATH=... repro infra install-root --target
       $MNT --source $STAGE --device /dev/nbdN --disko <json>
       --hostname <hn>.
    10. Write /etc/shadow entry (from TOML user.password_hash).
    11. umount in reverse order (esp before root); qemu-nbd
        disconnect.
    12. mv qcow2 to recipe build/ dir.

WALL-CLOCK ENVELOPE
===================

| Phase                    | Cold | Warm |
| ------------------------ | ---: | ---: |
| stage-de-rootfs.sh       |  5m  | 0s (cache) |
| relocate-nix-to-repro.sh |  3m  | 0s (cache) |
| disk apply (sgdisk+mkfs) | 15s  | 15s |
| rsync stage -> mnt       |  3m  |  3m |
| install-root grub+fstab  |  5s  |  5s |
| umount + qemu-nbd        |  3s  |  3s |
| **total (recipe build)** |  ~16 min | 146s |
| **boot smoke**           |   97s |  97s |

The 16-min cold + 2.5-min boot = 18.5 min total iteration is a
~5x speedup vs. the M9.R.36-49 60-90 min cycle.  The warm-recipe
cache hit case (full skip of the recipe `build` action) would be
~5 min if M9.R.50.x output-decl fix is closed.

REMAINING GAPS (M9.R.50.x FOLLOW-UPS)
=====================================

(1) Engine output-decl promotion.  Recipe declares
    extraOutputs=[build/reproos-installed.qcow2] but the engine
    cache HIT reports "executed" without copying to .repro/output/
    install/.  Need DSL output-decl shape fix; the artifact IS
    produced at the build/ path (boot smoke reads it from there).

(2) Boot-from-disk initramfs.  The seeded initrd is the iso
    recipe's live-boot initramfs (probes for
    /live/filesystem.squashfs); on the installed disk it falls
    through to a busybox rescue shell.  Full multi-user.target
    boot needs initramfs-tools / dracut / a custom switch-root
    initramfs.  Until this lands, the smoke proves bootability at
    the bootloader + kernel + rootfs layers (which is what the
    "build artifact is valid" claim requires), not full
    userspace startup.

(3) renderFstab + renderInstalledGrubCfg emit raw /dev/nbd0pN
    build-time device paths.  Should emit LABEL=ESP / LABEL=
    reproos-root (mkfs.* already pins those labels via disko).
    Closure: change two Nim funcs in infra_install_root.nim.

(4) Recipe DSL shape -- declare the kernel + initrd + DE binaries
    as `uses:` deps so the engine refingerprints when any of them
    changes (currently extraInputs is just script paths).  Closure:
    one DSL refactor.

(5) Schema validator -- migrate from the bash awk-based toml_get
    to the in-tree libs/nim-toml-serialization parser (proper TOML
    semantics + better error messages).  Closure: small Nim helper
    app at apps/reproos-image-config/.

MIGRATION NOTES
===============

What's deprecated (for SMOKE -- still works for end-user installs):
  - reproos-installer-autorun.service systemd unit + the boot-time
    autorun path through the QML wizard.
  - _m9r{36..49}_install*.sh + _m9r{40..46}_boot_installed.sh
    driver scripts (replaced by _m9r50_build.sh + _m9r50_boot_smoke.sh).
  - REPRO_INSTALLER_AUTORUN=1 env-var in recipes/reproos-iso/repro.nim.

What's kept:
  - All of libs/repro_cli_support/.../infra_install_root.nim
    (M9.R.41 install-time root-mirror).  The image recipe invokes
    the same code path via `repro infra install-root`.
  - All of libs/repro_profile/.../disk_apply.nim (M9.R.22b disko-
    style apply).
  - All of recipes/reproos-iso/scripts/stage-de-rootfs.sh +
    relocate-nix-to-repro.sh.  The image recipe inlines them.
  - The reproos-iso recipe.  ISO booting + wizard flow remain for
    actual end-user installs.
  - The reproos-installer wizard app.  In the M9.R.50.x follow-up
    plan, the wizard's final step will emit a TOML to /tmp/repro-
    installer-<pid>/auto-config.toml and `repro build recipes/
    reproos-image -- --config /tmp/.../auto-config.toml` to drive
    the same recipe.
